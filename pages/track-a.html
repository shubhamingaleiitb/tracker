A1. Operating Systems & Memory Months 1–3

When you deployed DeepSeek-OCR on RunPod and hit flash-attention issues, the root cause was a systems problem. Understanding OS fundamentals makes you dangerous in debugging.

Resources

ResourceTypeWhy It Matters
OSTEP (free online) Book Best OS textbook; focus on virtualization and concurrency chapters
MIT 6.S081 Labs Course Hands-on xv6 labs; do at least the first 4
Julia Evans' Zines Visual Bite-sized Linux/systems concepts for quick reference

What to Focus On

  • Virtual memory and paging — understand why models OOM and what's actually happening OSTEP Ch. 15
  • Processes vs threads — why vLLM uses multiprocessing, why not threads? OSTEP Ch. 26
  • I/O and file systems — why is loading model weights slow? What's mmap? OSTEP Ch. 36
  • Linux profiling tools — strace, perf, nvidia-smi, htop on your own running services Brendan Gregg's Linux Perf
  • Complete OSTEP Part 1 — Virtualization (Chapters 3–24)
  • Complete OSTEP Part 2 — Concurrency (Chapters 25–34)
  • MIT 6.S081 Lab 1: Xv6 and Unix utilities Lab page
  • MIT 6.S081 Lab 2: System calls Lab page
  • MIT 6.S081 Lab 3: Page tables Lab page
  • MIT 6.S081 Lab 4: Traps Lab page
  • Practice: Profile DeepSeek-OCR deployment with strace/nvidia-smi. Trace a single inference request from API call to GPU execution. Write up findings.

A2. Networking & Distributed Systems Basics Months 2–4

Every model serving setup involves networking. Your FastAPI + vLLM deployment on RunPod is a distributed system in miniature.

Resources

ResourceTypeWhy It Matters
Computer Networking: Top-Down Approach Book Chapters 1–5; understand TCP/IP, HTTP, DNS
High Performance Browser Networking Free Book Practical web networking — free online at hpbn.co
Hussain Nasser (YouTube) Video Excellent HTTP, TCP, gRPC explanations

What to Focus On

  • TCP/IP fundamentals — understand the protocols your APIs use HPBN: TCP
  • HTTP/2 and gRPC — the protocols used in model serving HPBN: HTTP/2
  • Latency analysis — where does time go in a network request?
  • Load balancing and connection pooling — relevant for scaling model serving
  • Basics of distributed consensus — understand trade-offs (CAP, Raft overview) Raft Visualization
  • DNS and how the internet works end-to-end HPBN: Latency & Bandwidth
  • Read Kurose & Ross Chapters 1–5 (or watch corresponding lectures) Online Lectures

A3. Low-Level PyTorch & GPU Programming Months 3–6

The engineering skill that most directly separates research engineers from application engineers. The people building DeepSeek-V3's MoE routing or Anthropic's inference stack understand computation at the GPU level.

Resources

ResourceTypeWhy It Matters
PyTorch Internals (ezyang) Blog Edward Yang's deep dive into PyTorch's architecture
CUDA Programming Guide Docs NVIDIA's official guide; read the first 5 chapters
Triton Tutorials Code Write GPU kernels in Python; lower barrier than raw CUDA
GPU Mode (YouTube) Video Community lectures on GPU programming for ML

What to Build Progressively

  • Write custom autograd functions for operations you use PyTorch Extending Guide
  • Implement a simple attention kernel in Triton Triton Attention Tutorial
  • Profile a model forward pass with PyTorch profiler — understand where time goes PyTorch Profiler Recipe
  • Read Flash Attention's code and understand the tiling strategy Flash Attention Repo
  • Experiment: compare eager mode vs torch.compile vs CUDA graphs torch.compile Tutorial
  • Read ezyang's PyTorch Internals blog series Blog
  • Read CUDA Programming Guide chapters 1–5 CUDA Guide
  • Complete all Triton getting-started tutorials Triton Tutorials
